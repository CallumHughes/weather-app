          Persists a manual display order. The body must contain every one of
          the signed-in user's favourite ids exactly once, top first; each
          favourite's `sortOrder` becomes its position in the list. If the id
          set does not match the user's current favourites (for example the list
          changed in another tab) the request is rejected with 409
          `FAVOURITES_OUT_OF_SYNC` — re-fetch and retry. Responds with the
          freshly ordered list. Requires a Better-Auth session cookie — see the
          Authentication page.
